# 代码规范

> **规则更新说明**：此文档为动态维护的代码规范集合，当项目中出现新的编码场景或需要补充编程规则时，应在此文档中添加相应规范条目。所有开发者在编写代码前应先阅读此文档，确保代码风格一致性。

## 注释规范
注释只标注关键部分，不要每一行代码都标注

## 代码修改规范
所有代码书写前应该先使用存在的函数，优先复用，并使用相同逻辑。
写代码的时候如果发现重复三次以上的书写同一种函数就把关键部分提取出来，封装成一个可调用的函数，降低耦合度。
修改代码之后不要影响其他功能的正常使用
确保写完之后进行测试，确保功能完全可用，且不影响其他功能正常使用

## 代码检查规范
提到A功能不好使但B功能好使，并且他们属于同一类型函数时，着重检查A、B代码之间的区别，确保A使用和B完全相似的逻辑结构。实在不行可以完全复制B的代码，重新书写A代码，测试成功之后，找到原有问题再进行修改

## 导入规范

### 1. 导入位置

所有 import 语句必须放在文件顶部，除非满足以下条件之一：
- **迫不得已**：为了避免循环导入
- **影响逻辑**：延迟导入不会改变程序逻辑

### 2. 导入顺序

按照以下顺序组织导入语句：

```python
# 1. 标准库导入
import os
import sys
import time
from datetime import datetime

# 2. 第三方库导入
import requests
from bs4 import BeautifulSoup
from loguru import logger

# 3. 本地模块导入
from config.common_config import db
from spider_modules.SpiderSession import SpiderSession
from utils.proxy_manager import proxy_manager
```

### 3. 禁止在函数或类内部导入

```python
# ❌ 错误：在函数内部导入
def some_function():
    import requests
    from config.common_config import db
    result = db.execute_sql("SELECT * FROM users", fetch="fetch")
    return result

# ✅ 正确：在文件顶部导入
import requests
from config.common_config import db

def some_function():
    result = db.execute_sql("SELECT * FROM users", fetch="fetch")
    return result
```

### 4. 延迟导入的例外情况

以下情况允许在函数或类内部导入：

#### 4.1 避免循环导入


## 数据库使用规范

### 1. 数据库操作统一使用 execute_sql 方法

所有数据库操作必须使用 `SQLiteDB` 类的 `execute_sql` 方法，不要直接使用 `cursor.execute()` 或其他底层方法。

### 2. execute_sql 方法签名

```python
def execute_sql(
    self,
    sql: str,
    params: Union[Tuple, List, Dict, None] = None,
    *,
    fetch: str = "none",  # none/fetch/fetch_one
    commit: bool = True
) -> Union[List[Dict], Dict, int, None]:
```

### 3. fetch 参数说明

`fetch` 参数控制结果获取方式，**仅支持以下三种值**：

| fetch 值 | 说明 | 适用场景 | 返回值 |
|---------|------|---------|--------|
| `"none"` | 不获取结果（默认） | INSERT、UPDATE、DELETE、DDL 操作 | INSERT 返回自增ID（整数），UPDATE/DELETE 返回受影响行数（整数），DDL 返回 None |
| `"fetch"` | 获取所有结果 | SELECT 查询多条记录 | 字典列表 `List[Dict]` |
| `"fetch_one"` | 获取单条结果 | SELECT 查询单条记录 | 单条字典 `Dict` 或 `None` |

**注意：** fetch 参数值必须使用下划线，不要使用驼峰命名（如 `"fetchone"` 是错误的）。

### 4. 使用示例

#### 4.1 INSERT 操作（fetch="none"）

```python
# 插入单条记录
result = db.execute_sql(
    "INSERT INTO users (name, age) VALUES (?, ?)",
    params=("张三", 25),
    fetch="none"
)
# result 返回自增 ID

# 插入多条记录（使用字典参数）
result = db.execute_sql(
    "INSERT INTO users (name, age) VALUES (:name, :age)",
    params={"name": "李四", "age": 30},
    fetch="none"
)
```

#### 4.2 SELECT 查询多条记录（fetch="fetch"）

```python
# 查询所有用户
users = db.execute_sql(
    "SELECT id, name, age FROM users WHERE age > ?",
    params=(20,),
    fetch="fetch"
)
# users 返回: [{"id": 1, "name": "张三", "age": 25}, {"id": 2, "name": "李四", "age": 30}]

# 遍历结果
for user in users:
    print(f"ID: {user['id']}, 姓名: {user['name']}, 年龄: {user['age']}")
```

#### 4.3 SELECT 查询单条记录（fetch="fetch_one"）

```python
# 查询单个用户
user = db.execute_sql(
    "SELECT id, name, age FROM users WHERE id = ?",
    params=(1,),
    fetch="fetch_one"
)
# user 返回: {"id": 1, "name": "张三", "age": 25} 或 None

if user:
    print(f"找到用户: {user['name']}")
else:
    print("用户不存在")
```

#### 4.4 UPDATE 操作（fetch="none"）

```python
# 更新用户信息
affected_rows = db.execute_sql(
    "UPDATE users SET age = ? WHERE id = ?",
    params=(26, 1),
    fetch="none"
)
# affected_rows 返回受影响的行数
```

#### 4.5 DELETE 操作（fetch="none"）

```python
# 删除用户
affected_rows = db.execute_sql(
    "DELETE FROM users WHERE id = ?",
    params=(1,),
    fetch="none"
)
# affected_rows 返回受影响的行数
```

### 6. 常见错误示例

#### 错误 1：fetch 参数值错误

```python
# ❌ 错误：使用了驼峰命名
result = db.execute_sql(
    "SELECT * FROM users WHERE id = ?",
    params=(1,),
    fetch="fetchone"  # 错误！
)

# ✅ 正确：使用下划线命名
result = db.execute_sql(
    "SELECT * FROM users WHERE id = ?",
    params=(1,),
    fetch="fetch_one"  # 正确
)
```

#### 错误 2：SELECT 查询未指定 fetch 参数

```python
# ❌ 错误：SELECT 查询未指定 fetch 参数，无法获取结果
result = db.execute_sql(
    "SELECT * FROM users WHERE id = ?",
    params=(1,)
)
# result 返回 None，无法获取查询结果

# ✅ 正确：指定 fetch 参数
result = db.execute_sql(
    "SELECT * FROM users WHERE id = ?",
    params=(1,),
    fetch="fetch_one"
)
# result 返回查询结果
```

#### 错误 3：INSERT/UPDATE/DELETE 指定了 fetch 参数

```python
# ❌ 错误：INSERT 操作不需要指定 fetch 参数
result = db.execute_sql(
    "INSERT INTO users (name, age) VALUES (?, ?)",
    params=("张三", 25),
    fetch="fetch"  # 不需要
)

# ✅ 正确：使用默认的 fetch="none"
result = db.execute_sql(
    "INSERT INTO users (name, age) VALUES (?, ?)",
    params=("张三", 25)
)
# 或者显式指定
result = db.execute_sql(
    "INSERT INTO users (name, age) VALUES (?, ?)",
    params=("张三", 25),
    fetch="none"
)
```

### 7. 完整示例

```python
from config.common_config import db

# 1. 插入数据
user_id = db.execute_sql(
    "INSERT INTO users (name, age, email) VALUES (?, ?, ?)",
    params=("张三", 25, "zhangsan@example.com"),
    fetch="none"
)
print(f"插入成功，用户ID: {user_id}")

# 2. 查询单条数据
user = db.execute_sql(
    "SELECT id, name, age, email FROM users WHERE id = ?",
    params=(user_id,),
    fetch="fetch_one"
)
if user:
    print(f"用户信息: {user}")

# 3. 查询多条数据
users = db.execute_sql(
    "SELECT id, name, age FROM users WHERE age > ?",
    params=(20,),
    fetch="fetch"
)
print(f"找到 {len(users)} 个用户")

# 4. 更新数据
affected = db.execute_sql(
    "UPDATE users SET age = ? WHERE id = ?",
    params=(26, user_id),
    fetch="none"
)
print(f"更新了 {affected} 条记录")

# 5. 删除数据
affected = db.execute_sql(
    "DELETE FROM users WHERE id = ?",
    params=(user_id,),
    fetch="none"
)
print(f"删除了 {affected} 条记录")
```

### 8. 注意事项

1. **fetch 参数必须使用下划线**：`fetch_one` 而不是 `fetchone`
2. **SELECT 查询必须指定 fetch 参数**：`fetch="fetch"` 或 `fetch="fetch_one"`
3. **INSERT/UPDATE/DELETE 使用默认 fetch="none"**：不需要显式指定
4. **使用参数化查询**：不要拼接 SQL 字符串，防止 SQL 注入
5. **检查返回值**：根据操作类型正确处理返回值（ID、行数、结果集等）

## 线程与并发规范

### 1. 线程安全

#### 1.1 使用互斥锁保护共享资源

```python
from PyQt5.QtCore import QMutex

# 全局互斥锁：防止并发操作
operation_mutex = QMutex()

def critical_operation():
    operation_mutex.lock()
    try:
        # 执行需要保护的操作
        result = some_shared_resource.modify()
    finally:
        operation_mutex.unlock()
    return result
```

#### 1.2 使用上下文管理器（推荐）

```python
from PyQt5.QtCore import QMutexLocker

def critical_operation():
    with QMutexLocker(operation_mutex):
        # 执行需要保护的操作，自动加锁和解锁
        return some_shared_resource.modify()
```

### 2. 线程间通信

#### 2.1 使用信号槽机制

```python
from PyQt5.QtCore import QObject, pyqtSignal, QThread

class WorkerThread(QThread):
    # 定义信号
    log_signal = pyqtSignal(str)  # 日志信号
    progress_signal = pyqtSignal(int)  # 进度信号
    finished_signal = pyqtSignal(bool, str)  # 完成信号：(是否成功, 消息)
    
    def run(self):
        try:
            for i in range(100):
                # 发送进度信号
                self.progress_signal.emit(i)
                
                # 执行工作
                result = do_work(i)
                
                # 发送日志信号
                self.log_signal.emit(f"处理项目 {i}: {result}")
                
            # 发送完成信号
            self.finished_signal.emit(True, "处理完成")
        except Exception as e:
            self.finished_signal.emit(False, str(e))
```

#### 2.2 在主线程中更新UI

```python
class MainWindow(QWidget):
    def __init__(self):
        super().__init__()
        self.worker = WorkerThread()
        # 连接信号到槽
        self.worker.log_signal.connect(self.update_log)
        self.worker.progress_signal.connect(self.update_progress)
        self.worker.finished_signal.connect(self.on_finished)
    
    def update_log(self, message):
        # 在主线程中更新UI
        self.log_text.append(message)
    
    def update_progress(self, value):
        # 在主线程中更新进度条
        self.progress_bar.setValue(value)
    
    def on_finished(self, success, message):
        if success:
            QMessageBox.information(self, "成功", message)
        else:
            QMessageBox.warning(self, "错误", message)
```

### 3. 异步操作

#### 3.1 避免阻塞主线程

```python
# ❌ 错误：在主线程中执行耗时操作
def on_button_click(self):
    result = long_running_operation()  # 阻塞主线程
    self.update_ui(result)

# ✅ 正确：使用线程执行耗时操作
def on_button_click(self):
    self.worker = WorkerThread()
    self.worker.finished.connect(self.update_ui)
    self.worker.start()
```

#### 3.2 使用线程池处理并发任务

```python
import concurrent.futures
from threading import Lock

# 线程安全的计数器
counter_lock = Lock()
counter = 0

def process_item(item):
    global counter
    result = expensive_operation(item)
    
    with counter_lock:
        counter += 1
        print(f"已处理 {counter} 个项目")
    
    return result

# 使用线程池
with concurrent.futures.ThreadPoolExecutor(max_workers=4) as executor:
    futures = [executor.submit(process_item, item) for item in items]
    results = [future.result() for future in concurrent.futures.as_completed(futures)]
```

## 错误处理与日志规范

### 1. 异常处理

#### 1.1 基本异常处理

```python
from loguru import logger

def process_data(data):
    try:
        # 可能抛出异常的代码
        result = risky_operation(data)
        return result
    except SpecificException as e:
        # 捕获特定异常
        logger.error(f"处理数据时发生特定异常: {e}")
        return None
    except Exception as e:
        # 捕获所有其他异常
        logger.error(f"处理数据时发生未预期异常: {e}", exc_info=True)
        raise
    finally:
        # 无论是否发生异常都会执行的代码
        cleanup_resources()
```

#### 1.2 使用上下文管理器

```python
class DatabaseConnection:
    def __enter__(self):
        self.connection = create_connection()
        return self.connection
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        if exc_type:
            logger.error(f"数据库操作发生异常: {exc_val}")
        self.connection.close()

# 使用上下文管理器
def process_with_database():
    with DatabaseConnection() as conn:
        # 使用数据库连接
        result = conn.execute("SELECT * FROM users")
        return result
    # 连接自动关闭
```


### 2. 前端与后端任务管理区别

#### 2.1 概念区分

- **前端任务管理**：指`index.html`中的任务管理界面，用于展示和管理用户在Web界面中创建和监控的任务
- **后端任务管理**：指数据库中的`task`表，用于存储任务的基本信息、状态和执行结果

#### 2.2 数据流向

```
前端任务管理界面 -> 创建/启动任务 -> 后端API -> 写入任务信息 -> 数据库task表
数据库task表 -> 任务状态更新 -> 后端API -> 实时推送状态 -> 前端任务管理界面
```

#### 2.3 任务ID处理

```python
# ✅ 正确：前端任务ID与后端任务ID一致
```

#### 3.2 配置验证

```python
def validate_config(key, value, expected_type):
    """验证配置值是否符合预期类型"""
    try:
        if expected_type == bool:
            return str(value).lower() in ("true", "1", "yes")
        elif expected_type == int:
            return int(value)
        elif expected_type == float:
            return float(value)
        elif expected_type == str:
            return str(value)
        else:
            return value
    except (ValueError, TypeError):
        logger.warning(f"配置值类型转换失败 | key: {key} | value: {value} | expected_type: {expected_type}")
        return None

# 使用示例

    "max_retries", 
    config_manager.get_or_set_config("max_retries", "3"), 
    int
) or 3  # 提供默认值
```
