# 添加新任务功能指南

## 概述

本指南详细说明如何在系统中添加一个新的任务功能，以核价任务为基准，以虎扑帖子列表任务为例。整个过程包括后端任务函数封装、前端界面添加和功能测试。

## 第一步：分析现有任务结构

### 1.1 核价任务结构分析

核价任务的核心组件：
- **任务包装函数**：`modify_price_task_wrapper` (在 `temu_modules/temu_func_wrapper.py` 中)
- **核心实现函数**：`all_modify_price` (在 `temu_modules/temu_function/modify_price.py` 中)
- **前端任务卡片**：在 `templates/index.html` 中的任务卡片
- **前端任务模态框**：在 `static/js/main.js` 中的 `openTaskModal` 函数
- **任务路由处理**：在 `api/server_routes/task_routes.py` 中

### 1.2 虎扑帖子列表任务结构分析

虎扑帖子列表任务的核心组件：
- **任务包装函数**：`process_hupu_posts_chunk` (在 `spider_modules/hupu_func_wrapper.py` 中)

## 第二步：创建新任务的核心实现函数

### 2.1 创建核心实现文件

首先，在适当的模块目录中创建核心实现文件。例如，如果是一个爬虫任务，则在 `spider_modules/` 目录下创建新文件；如果是Temu相关任务，则在 `temu_modules/temu_function/` 目录下创建新文件。

### 2.2 实现核心功能

```python
# 示例：new_task_function.py
from loguru import logger
import time

def new_task_function(param1, param2, param3=None):
    """
    新任务的核心实现函数
    
    Args:
        param1: 参数1说明
        param2: 参数2说明
        param3: 参数3说明（可选）
    
    Returns:
        dict: 包含执行结果的字典
    """
    try:
        logger.info(f"开始执行新任务 | param1: {param1} | param2: {param2}")
        
        # 实现任务逻辑
        result = []
        for i in range(10):  # 示例逻辑
            logger.info(f"处理项目 {i+1}")
            # 执行具体操作
            item = process_item(param1, param2, i)
            result.append(item)
            
            # 可选：添加休息时间
            time.sleep(0.5)
        
        logger.info(f"新任务执行完成 | 处理数量: {len(result)}")
        return {"code": 1, "msg": "任务执行成功", "data": {"items": result, "count": len(result)}}
    except Exception as e:
        logger.error(f"新任务执行失败: {e}")
        return {"code": -1, "msg": f"任务执行失败: {str(e)}", "data": {}}
```

## 第三步：创建任务包装函数

### 3.1 在适当的包装器文件中添加任务包装函数

```python
# 在 temu_modules/temu_func_wrapper.py 或 spider_modules/hupu_func_wrapper.py 中添加

def new_task_wrapper(
    uid,
    param1,
    param2,
    param3=None,
    login_type: str = "auto",
    reload_cookies: bool = False,
    headless: bool = True,
    main_task_id=None,
    auto_close: bool = True,
    window_size: tuple[int, int] = (1920, 1080)
):
    """
    新任务的包装函数
    
    Args:
        uid: 用户ID
        param1: 参数1说明
        param2: 参数2说明
        param3: 参数3说明（可选）
        login_type: 登录类型
        reload_cookies: 是否重新加载cookies
        headless: 是否无头模式
        main_task_id: 主任务ID
        auto_close: 是否自动关闭浏览器
        window_size: 窗口大小
    
    Returns:
        dict: 包含执行结果的字典
    """
    # 如果是Temu相关任务，需要登录
    if hasattr(temu_login_warpper, '__call__'):
        login_result = temu_login_warpper(
            uid, 
            login_type=login_type, 
            reload_cookies=reload_cookies, 
            headless=headless,
            auto_close=auto_close, 
            window_size=window_size
        )
        if login_result["code"] != 1:
            return {
                "code": -1, 
                "msg": f"{login_result['msg']}", 
                "remarks": login_result["remarks"], 
                "data": {}
            }

        shop_info = login_result["data"]
        shop_abbr = shop_info.get("shop_abbr", "未知店铺")
        mall_id = shop_info.get("mall_id", 0)
    else:
        shop_abbr = "系统任务"
        mall_id = 0
    
    try:
        # 调用核心实现函数
        result = new_task_function(
            param1=param1,
            param2=param2,
            param3=param3,
            shop_abbr=shop_abbr,
            mall_id=mall_id,
            main_task_id=main_task_id,
        )
        return result
    except Exception as e:
        logger.error(f"❌ 店铺{shop_abbr} 新任务失败：{str(e)[:200]}")
        return {
            "code": -1, 
            "msg": f"新任务失败：{str(e)[:200]}", 
            "data": {}, 
            "remarks": str(e)[:500]
        }
```

## 第四步：添加任务路由处理

### 4.1 在 task_routes.py 中添加任务路由

```python
# 在 api/server_routes/task_routes.py 中添加

@router.post("/api/create_new_task", dependencies=[Depends(verify_token)])
def create_new_task_api(request: NewTaskRequest):
    """创建新任务"""
    try:
        # 获取任务参数
        task_params = {
            "uid": request.uid,
            "param1": request.param1,
            "param2": request.param2,
            "param3": request.param3,
            "login_type": request.login_type,
            "reload_cookies": request.reload_cookies,
            "headless": request.headless,
            "auto_close": request.auto_close,
            "window_size": request.window_size,
        }
        
        # 创建任务
        task_id = get_task_log_manager().add_task(
            target_func=new_task_wrapper,
            **task_params,
            task_group="新任务组",
            is_main_task=1,
        )
        
        return {
            "success": True,
            "data": {"task_id": task_id},
            "message": "新任务创建成功"
        }
    except Exception as e:
        logger.error(f"创建新任务失败: {e}")
        return JSONResponse(
            status_code=500,
            content={"success": False, "error_msg": str(e)}
        )

# 添加请求模型
from pydantic import BaseModel, Field

class NewTaskRequest(BaseModel):
    uid: int = Field(..., description="用户ID")
    param1: str = Field(..., description="参数1")
    param2: str = Field(..., description="参数2")
    param3: str = Field(None, description="参数3（可选）")
    login_type: str = Field("auto", description="登录类型")
    reload_cookies: bool = Field(False, description="是否重新加载cookies")
    headless: bool = Field(True, description="是否无头模式")
    auto_close: bool = Field(True, description="是否自动关闭浏览器")
    window_size: tuple[int, int] = Field((1920, 1080), description="窗口大小")
```
